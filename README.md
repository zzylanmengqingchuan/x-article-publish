# X Article Publish Helper

一个专注于 **X Articles（x.com/compose/articles）** 的 Chrome 扩展。  
目标很明确：把 Markdown 内容快速粘贴到 X 文章编辑器，并尽可能自动化处理图片上传流程。

## 这个插件解决了什么问题

在 X 文章编辑器中，普通 Markdown 图片语法（如 `![alt](https://...)`）不会直接变成已上传的媒体块。  
手动发文时，通常需要反复复制、粘贴、上传图片，长文发布体验比较低效。

本插件提供：

- Markdown 文本导入（粘贴或导入 `.md` 文件）
- 自动打开 X Articles 页面
- 将 Markdown 转为适合编辑器的富文本粘贴内容
- 对代码块 / 图片占位符进行后处理
- 通过插件后台下载图片并尝试触发编辑器原生媒体上传链路

## 功能特性

- 仅支持 X Articles 页面（`https://x.com/compose/articles`）
- 支持从弹窗导入本地 `.md/.markdown/.txt`
- 支持直接粘贴 Markdown
- 支持图片 URL 的自动下载与插入尝试
- 支持基本进度提示（开始、处理中、完成/失败）

## 安装方式（开发者模式）

1. 打开 Chrome 浏览器，进入 `chrome://extensions/`
2. 开启右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择本项目目录（`x-article-publish`）
5. 安装完成后，建议固定到工具栏便于使用

## 使用步骤

1. 登录 X 账号
2. 点击插件图标，粘贴 Markdown（或点击 `Import .md` 导入文件）
3. 点击 `Open X and Publish`
4. 插件会跳转到 X Articles 页面
5. 进入真正的文章编辑态（如页面出现“撰写”按钮，需要先点击进入编辑器）
6. 等待自动粘贴和后处理完成

## 注意事项（请务必阅读）

- 图片链接必须可公开访问（无鉴权、无过期签名、无防盗链限制）
- X 编辑器是动态页面，结构变更后可能影响自动化稳定性
- 若你同时安装了其他 Markdown 粘贴类扩展，可能会产生冲突，建议先禁用同类扩展再测试
- 首次使用或修改代码后，请到 `chrome://extensions` 手动 `Reload` 当前扩展

## 常见问题

### 1) 文本可以粘贴，但图片没有上传

优先检查：

- 图片 URL 是否能在浏览器直接打开
- 是否有跨域/鉴权限制导致后台下载失败
- 是否已进入 X 的真实编辑器（不是列表页）
- 是否存在同类扩展冲突

### 2) 控制台出现 React minified error（如 #418 / #423）

这类报错通常来自页面内部状态与外部注入操作时机冲突，不一定只由本插件单独触发。  
建议结合插件日志和页面状态一起排查，不要只看单一错误码。

### 3) 成功提示一直不消失

已在当前版本修复：成功/警告提示会自动隐藏。
